// How information flows

The system is built around one reality: during enumeration/exploitation you are time-boxed and your brain is in “hunt mode”. So the first step is capture, not organization.

  1. Capture everything in 00_Inbox (commands, URLs, quick thoughts, payload trials).
  2. Promote only the signal into a target note in 10_Machines (clean narrative + evidence).
  3. Extract repeatable stuff into 20_Enumeration, 30_Exploits, 40_CheatSheets.
  4. Archive old/solved content to keep the active surface minimal.

Result: every machine is a finished story, and every story produces reusable building blocks.

// Design principles

  • Low friction: if it takes effort, you won’t do it during a real run.
  • Single source of truth: one machine note per target, not fragments.
  • Reusable by default: checklists & one-liners evolve from real fights.
  • Evidence-first: commands + outputs + proof are always nearby.
  • Redaction-ready: structure is public; secrets are not.

// Naming & evidence conventions

Consistency beats perfection
Files

Use timestamp + host + tool + intent.

2025-12-18__10.10.10.10__nmap__full.txt
Screenshots

Name by step, not by emotion.

07_foothold__webshell.png
Links

Store stable references in 90_Resources, not inside every note.

90_Resources/Links.md

// What I never publish

  • Real credentials, tokens, API keys
  • Private hostnames / internal domains
  • Client identifiers, IP ranges, scope docs
  • Anything that enables unauthorized access
Public version idea

Share the structure, workflows, and redacted templates. Replace sensitive values with placeholders like TARGET, USER, PASS.

OSCP_Exam_Checklist.md
# OSCP Flow (Quick)

## 1) Initial
- [ ] Nmap full
- [ ] HTTP/HTTPS quick recon
- [ ] SMB/LDAP/Kerberos checks (if open)

## 2) Web
- [ ] Tech fingerprint
- [ ] Content discovery (ffuf)
- [ ] Auth checks / default creds
- [ ] LFI/RFI/SQLi/RCE

## 3) AD
- [ ] Users/groups/SPNs
- [ ] Kerberoast/ASREPRoast
- [ ] Shares + ACL paths

## 4) PrivEsc
- [ ] Linux: sudo, SUID, cron, capabilities
- [ ] Windows: token/privs, services, scheduled tasks

## 5) Reporting
- [ ] Attack path
- [ ] Evidence
- [ ] Fixes
